### Introduction

데이터베이스 관리 시스템(DBMS)의 역사는 1970년대 초 IBM의 에드거 F. 커드(Edgar F. Codd) 박사가 발표한 논문에서 시작되었습니다. 커드 박사는 기존의 계층형(Hierarchical) 모델(IMS)과 네트워크(Network) 모델(Codasyl)이 가지고 있던 데이터 중복성(Data Redundancy) 문제와 낮은 데이터 독립성 문제를 해결하기 위해 수학적 이론에 기반한 새로운 모델을 제안했는데, 이것이 바로 관계형 모델(Relational Model)입니다.

이 모델의 핵심 목표는 사용자가 데이터의 물리적 저장 방식이나 접근 경로에 대해 전혀 신경 쓰지 않고, 오직 논리적 데이터 구조와 원하는 결과에만 집중할 수 있도록 데이터 독립성(Data Independence)을 극대화하는 것이었습니다.

---

### Relational Model Definitions

관계형 모델은 데이터를 2차원 테이블 형태로 표현하며, 이는 수학적 개념인 릴레이션(Relation)에 기초를 둡니다.

데이터베이스에서 하나의 테이블을 지칭하는 릴레이션, 특정 릴레이션의 이름과 그 릴레이션이 포함하는 애트리뷰트(Attribute)들의 집합인 스키마는 테이블의 헤더입니다. 릴레이션의 열은 애트리뷰트이고 행은 튜플입니다.

관계형 릴레이션은 다음과 같은 중요한 수학적 특성을 가집니다.

릴레이션 내의 튜플들은 순서가 없습니다. 튜플의 물리적 저장 순서는 논리적 의미에 영향을 미치지 않습니다. 릴레이션 내의 애트리뷰트들 역시 순서가 없습니다. 이는 애트리뷰트 이름으로 식별되기 때문입니다. 각 애트리뷰트의 값은 반드시 원자적(Atomic)이어야 합니다. 즉, 하나의 셀에는 더 이상 분해할 수 없는 단일 값만 포함되어야 합니다. 이것이 제1 정규형(1NF)의 기본 전제입니다.

### Integrity Constraints

무결성 제약 조건(IC)은 데이터베이스에 저장된 데이터의 정확성과 일관성을 유지하기 위해 DBMS가 자동으로 시행하는 규칙입니다.

릴레이션에서 튜플을 고유하게 식별하는 데 사용되는 하나 이상의 애트리뷰트 집합 키와 릴레이션의 모든 튜플을 고유하게 식별할 수 있는 최소한의 애트리뷰트 집합인 후보 키, 여러 후보 키 중 설계자가 주요 식별자로 선택한 후보 키인 기본 키가 있을 때 기본 키를 구성하는 애트리뷰트 값은 널(NULL)을 가질 수 없습니다. 만약 널 값이 허용된다면 튜플을 고유하게 식별할 수 없게 되어 무결성이 깨집니다.

다른 릴레이션의 기본 키를 참조하는 애트리뷰트 집합인 외래 키는 릴레이션 간의 논리적 연결을 설정할 수 있고 참조하는 릴레이션에 실제로 존재하는 기본 키 값과 일치하거나 모든 외래 키 애트리뷰트가 널(NULL)이어야 참조 무결성 제약 조건을 만족합니다.

만약 참조 무결성이 위반될 때 CASCADE (함께 삭제/갱신), SET NULL, SET DEFAULT 또는 RESTRICT 중 하나를 선택하여야 합니다.

---

### Data Independence

관계형 모델의 가장 큰 장점은 데이터 독립성을 제공하여 애플리케이션 개발을 단순화하고 유지보수 비용을 절감한다는 것입니다.

물리적 데이터 독립성에 따라 데이터베이스의 물리적 저장 구조나 접근 방법이 변경되더라도 논리적 스키마(테이블 구조)와 애플리케이션 프로그램은 영향을 받지 않습니다. 그리고 논리적 데이터 독립성에 따라 데이터베이스의 논리적 스키마가 변경되더라도 (예: 테이블에 새로운 열을 추가하거나 기존 테이블을 분할해도) 사용자 뷰(View)를 통해 기존 애플리케이션 프로그램이 영향을 받지 않도록 보호할 수 있습니다. 뷰는 기본 테이블에 대한 SELECT 질의의 결과로 정의됩니다.
