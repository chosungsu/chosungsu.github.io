### Introduction

데이터베이스는 구조화된 데이터의 집합입니다. 일반적으로 '레코드' (전통적으로 디스크에 저장되는 대량의 데이터) 및 레코드 간의 관계로 구성됩니다. DBMS는 데이터베이스를 생성, 조작, 접근하기 위한 시스템입니다. 데이터 컬렉션 앞에 위치하여 애플리케이션의 데이터 접근을 중재합니다. DBMS의 목표는 데이터의 저장과 접근을 단순화하는 것입니다.

---

### Mafia Example

#### 파일 시스템 사용 시의 문제점

단순 텍스트 파일 사용 시 엔티티와 관계를 표현하기 어렵다는 점과 파일 단위가 아닌 낮은 세분성으로 접근 자격을 통제하기 어렵다는 점이 있습니다. 원하는 정보를 찾기 위해 별도의 절차적 프로그램을 작성하고 파일 전체를 스캔해야 해서 실시간성 보장 어렵습니다.

모두 실행되거나 아무것도 실행되지 않도록 보장해야 하는 원자성, 데이터가 항상 유효한 상태임을 보장해야 하는 일관성, 파일에 동시에 기록할 때 서로의 작업을 덮어쓰지 않도록 보호해야 하는 격리성, 컴퓨터 충돌 시에도 데이터가 손실되거나 뒤섞이지 않도록 하는 영속성이 보장되어야 합니다.

### Fundamental Concepts

DBMS는 위의 문제들을 해결하기 위해 컴퓨터 과학의 여러 분야(언어, 이론, OS, 동시성, 최적화 등)의 핵심 개념을 통합합니다.

#### Representing Data

데이터 모델은 데이터를 구조화하는 일관된 방식을 제공하여, 일관성, 공유, 효율적인 접근을 가능하게 합니다.

데이터 구성을 설명하는 구성 요소 집합으로 테이블, 그래프 등등이 해당하는 데이터 모델과 주어진 데이터 모델을 사용하여 특정 데이터 컬렉션을 설명한 개념/논리 스키마, 데이터의 물리적 조직인 디스크의 데이터와 파일은 물리적 스키마라고 합니다.

#### Transactions

트랜잭션은 원자적으로 발생해야 하는 행동들을 그룹화하는 방법입니다. 동시성 및 장애 발생 시에도 애플리케이션에 대한 데이터 보장을 제공합니다.
